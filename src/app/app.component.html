Instructions

We would like you to make a small application which calls three of our APIs found on developer.parliament.uk

The three APIs we are going to be asking you to call are:

  • Members API- this API supplies Member data (both Commons and Lords)
  • Commons votes- this API supplies vote data for Commons
  • Lords votes - this API supplies vote data for Lords

The task is:

  • Replace these instructions with a series of components that do the following
  • Call the Members API and show a paged list of Members
  • Add a filter at the top of the list which enables a user to filter Members based on their name and/or House
  • Show details about Members on cards
  • You can use similar styling to our styling - but please do not just copy it. We want to see how you might present this data
  • If a user clicks on a Member card then use the Votes APIs to show the last 3 votes the Member participated in, and show some relevant details

Members API

  • You can search Members using our search endpoint - /api/Members/Search
  • This has a Name and House parameter
  • When you receive data about a Member you can determine what House they sit in with the property "member.latestHouseMembership.house" 1 for Commons and 2 for Lords

Votes APIs

  • The contracts for Commons and Lords votes are the same, but they have separate endpoints
  • If a Member has House 1 then you need to use the Commons API
  • If a Member has House 2 then you need to use the Lords API
  • Please investigate the votes contracts to determine the best endpoint to call
